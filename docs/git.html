Git cheatsheet

Getting help

git help                Show git flags and common subcommands
git help help           Help for help
git help -g             *LIST HELP GUIDES*
git help git            All subcommands and help for git command
git help <command>      Show help for <command>
git <command> --help    Show help for <command>
man git-<command>       Show help for <command>
git <command> -h        Abbreviated <command> help
git <command> --help-all        Show hidden options if they exist, such as git
                                init --separate-git-dir

If using fish shell, type “git help” (with the space) then tab

Configuration

git config

git config –global alias.co checkout Alias a git command git config –global alias.unstage ‘reset HEAD –’ And another git config –global alias.visual ‘!gitk’ External commands require “!”

Log

git log Commit history -p Show patch introduced by each commit –stat Show stats about # of lines changed in each file –pretty= Pretty-print in the given , which is one of oneline, short, medium, full, fuller, email, raw, format:STRING, and tformat:STRING –all Show commits from all refs. Apparently the current branch is the only one shown by default. –graph ASCII graph of branch and merge history. Use in combination with --all. –decorate Print references (local and remote) next to commits –abbrev-commit Shortened SHA-1. –name-status Show filenames and change status for each commit. Status codes are the same as for –diff-filter –oneline Equiv. to –pretty=oneline –abbrev-commit -S STRING Filter commits to those in which STRING’s # of occurences have changed (e.g., function definition) –pickaxe-regex STRING passed to -S is POSIX extended regex -G REGEX Filter commits to those with added/removed lines containing REGEX, even if # occurences hasn’t changed (e.g., if REGEX was both added and removed on a line -G and not -S will match it). –pickaxe-all Show all changes for commit rather than just changes for files(s) containing the changes matched by -G or -S. Only seems to work in combination with -p. - Limit to commits –since= Filter out commits before date –until= Filter out commits after date .. Show commits on that don’t exist on . Can be used with remote-tracking branches to examine work before merging it into a branch.

git log --oneline --graph --all Visualize your git history w/ branches

Optionally truncate at beginning (ltrunc), middle (mtrunc), or end (trunc) if output is longer than N columns %>(N): Subsequent placeholder occupies N spaces, padding on left Commit-based placeholders: %n: newline %h: abbreviated commit hash %an: author name %ar: author date, relative %cn: committer name %cr: commiter date, relative %s: subject %b: body

Diff

git diff Diff b/w working tree and index –cached | –staged Diff b/w index and prior commit

Status

git status Show current branch, its status relative to its remote-tracking branch, which changes are staged, and untracked files untracked files, and which changes are staged -s Shorthand git status, with ?? for untracked, A for newly tracked (“added”), " M" for modified-unstaged, “M” for modified-staged, MM for staged and unstaged modifications. -b In short status, show branch info -v[v] Show staged [and unstaged] changes additionally, like git diff –cached [mixed with git diff] output

Show

git show Show various objects Show tag message (if one exists) and referenced object Show log message and diff

Init

git init Initialize a new repository in 
--bare      Bare/remote repository: no working dir, history stored at
            repository root rather than in .git within root. Customarily
            given a `.git` extension
--separate-git-dir <gitdir>     Separate the .git dir from the working
                                directory. <dir> is now the working tree
                                containing a .git file pointing to <gitdir>.
                                If working tree contains a .git directory,
                                it will be moved to <gitdir>.

Add

git add Track or stage changes in git add -A Track or stage changes for all files

Rm

git rm Stage file for untracking and deletion. File globbing can be used in . -f Forcing needed when file is modified or staged –cached Untrack but don’t delete file

Mv

git mv SRC DST Rename SRC to DST. Equiv. to mv SRC DST; git rm SRC; git add DST. Git will record this as a rename. ## Commit

git commit Record the index in the .git repository -m Inline commit message -v Include diffs in editor for ref when writing message -a Stage all unstaged changes before committing –amend Replace previous commit with current staging area

Reset

git reset [] Unstage if specified, else reset the index

Remote

git remote Show remote servers by shortname -v Additionally show repository locations (typically URLs) git remote show More info on remote git remote add Add remote for the repository at . Then git fetch will create and update remote-tracking branches / git remote rm Remove remote git remote rename Rename remote

Ls-remote

git ls-remote List remote references (e.g., branches)

Fetch

git fetch [ | -all] Fetch branches and/or tags (“refs”) from or all remotes, along with the objects needed to complete their histories. Update remote-tracking branches to reflect new state of remote branches. –unshallow Fetch the full repository history. Needed e.g., with any vim-plug-downloaded repository, because vim-plug only shallow clones.

Pull

git pull Fetch and merge from the upstream remote-tracking branch

Push

git push Update remote using local refs, while sending objects necessary to complete the given refs. By default the current branch is pushed to its upstream. Note that pushing requires write access to a remote repository and that nobody has pushed work that you have not yet merged. -u Push branch to remote and add a remote-tracking branch Push branch to remote –tags Push all tags to remote Push tag to remote –delete
